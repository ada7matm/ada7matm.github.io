Sass


Sass es un preprocesador de css. Nos permite escribir estilos de forma un poco diferente a css, que luego son procesados y convertidos a css de modo tal que el navegador los pueda interpretar. Al no regirse por la sintaxis de css, nos permite hacer otras cosas que son imposibles de hacer con puro css.


Instalación

Sass no se instala directamente. En cambio, necesitamos instalar primero una aplicación que nos permita instalarlo. Para eso vamos a bajar Node.js. Una vez que lo tenemos instalado, en la consola de Git Bash, introducimos el siguiente comando:


npm install -g sass



Cuando termine de descargarse e instalarse, podemos probar que efectivamente esté instalado con:


sass --version



Y debería aparecer la versión que tenemos en el sistema.


Escuchando archivos


Para que Sass convierta automáticamente los archivos con extensión .scss a archivos .css, tenemos que informarle cúales son dichos archivos a los que tiene que prestar atención, y a dónde queremos que guarde los archivos compilados.


Podemos hacer esto de dos formas, o diciéndole que escuche un archivo en particular:


sass --watch styles.scss styles.css



donde styles.css es el archivo sass y styles.css es el archivo al que queremos que compile nuestro código. Este comando tenemos que ejecutarlo desde Git Bash o desde la consola de Visual Studio, teniendo cuidado de estar paradas en la carpeta del proyecto.


Los nombres de los archivos son rutas relativas, por lo tanto, van a depender desde que carpeta lo estamos ejecutando. A su vez, tenemos que poner la ruta completa hacia el archivo, suponiendo que el archivos styles.css esté en una carpeta css, tenemos que poner:


sass --watch styles.scss css/styles.css



Otra forma de escuchar archivos es darle la instrucción a sass de que escuche carpetas directamente. En ese caso, va a buscar todos los archivos no parciales con extensión .scss que estén en la misma y convertilos a sus equivalentes archivos css en la carpeta de destino. Esto lo indicamos así:


sass --watch origen:destino



Donde origen y destino son los nombres de las carpetas respectivas. Si por ejemplo queremos que escuche el directorio styles y compile los archivos en la carpeta styles/css, lo indicamos de la siguiente manera:


sass --watch styles:styles/css




Variables


Una de las cosas que podemos hacer con Sass es utilizar variables. Supongamos que en nuestras páginas usamos tres tipos de tipografías, que se comparten entre muchos elementos. Cuando terminamos de maquetarla, la diseñadora gráfica nos diciéndole que necesitamos cambiar una de estas tipografías por otra. Si sucede esto, nos tocaría cambiar individualmente en cada uno de los elementos que utilicen esa fuente!


Por suerte podemos evitar esto con Sass. Sass nos permite crear variables, que son etiquetas que tienen asignadas un valor, por ejemplo, cierta tipografía. En vez de utilizar directamente esta fuente, podemos utilizar la variable. Cada elemento que utilice esta variable luego es convertido a css utilizando el valor que contiene. Por eso, si luego tenemos que cambiar nuestra tipografía, simplemente cambiamos el valor de variable, y todos los elementos que la estén utilizando se actualizarán automáticamente.


Una variable se declara de la siguiente forma:


$nombre-variable: valor;



Por ejemplo:


$main-font: "Roboto";



Y se utiliza simplemente llamándola con su nombre, anteponiendo el signo $, por ejemplo


h1 {
 font-family: $main-font;
}



Esto se traduce automática a css así:


h1 {
 font-family: "Roboto";
}


Lo bueno de las variables es que al tener un nombre, le dan un valor más semántico a nuestros estilos, es decir, es más fácil entender qué es lo que estamos estilando porque los valores tienen un cierto significado más allá de su valores númericos o de texto.


Dentro de una variable podemos guardar cualquier tipo de valores que irían dentro de propiedades css, por ejemplo:

$main-font-size: 15px; $button-border: 2px solid $main-color; $column-padding: 20px 30px; $background-transition: transition 1s ease-in 0.1s;




Selectores anidados


Otra de las cosas buenas que tiene Sass es que nos permite anidar selectores. Esto nos da una visualización más clara y evidente de la jerarquía de los elementos. Para anidar un selector dentro de otro, simplemente tenemos que declararlo dentro de las llaves de este:


main {
 width: 600px;

 article {
  width: 300px;
 }
}



Esto se traduce a css de la siguiente manera:


main { width: 600px; }
main article { width: 300px; }



Podemos anidar todos los selectores que queramos dentro de un selector, por ejemplo:


main {
 width: 600px;

 h1 {
  font-family: $main-font;
 }

 h2 {
  font-family: $secondary-font;
 }

 article {
  width: 300px;
 }
}



Que convertido en css nos queda como:


main { width: 600px; }
main h1 { font-family: "Roboto"; }
main h2 { font-family: "Montserrat"; }
main article { width: 300px; }



A su vez, podemos anidar selectores dentro de selectores todos los niveles que queramos. Si bien esto es perfectamente válido, nos es demasiado conveniente anidar más de 3 o 4 niveles como mucho, ya que de otra forma nos queda una jerarquía muy rígida, y si cambiamos de orden algo en el html, nuestros estilos se van a romper ya que nuestra jerarquía no aplica más.


Por ejemplo, supongamos que tenemos en un archivo sass declarado lo siguiente:


main {
 section {
  .column {
   .news {
    width: 300px;
    font-family: $news-font;
   }
  }
 }
}



Que se corresponde con el siguiente html:


<main>
 <section>
  <div class="column">
   <div class="news"></div>
  </div>
 </section>
</main>



Si ahora quisieramos utilizar .news en otro lugar, por ejemplo así:


<main>
 <section>
  <div class="column">
   <div class="news"></div>
  </div>
 </section>
 <section class="secondary-news">
  <div class="news"></div>
  <div class="news"></div>
  <div class="news"></div>
 </section>
</main>



Los estilos que declaramos para .news no se van a aplicar a los que se encuentran dentro de .secondary-news, ya que en la jerarquía que declaramos en nuestro archivo sass, necesitamos que esté dentro de un elemento .column para que aplique. Por eso, para casos así, lo mejor es declarar una jerarquía más flexible, como podría ser:


main {
 .news {
  width: 300px;
  font-family: $news-font;
 }
}



Si usamos el selecto & anidado, lo que estamos haciendo es referenciar al elemento padre. Esto nos sirve para utilizar en conjunto con pseudo-selectores, por ejemplo:


h1 {
 color: grey;

 &:hover {
  color: red;
 }
}



Lo que se compila como:


h1 {
 color: grey;
}
h1:hover {
 color: red;
}




Media queries


Podemos anidar media queries directamente dentro de selectores, sin necesidad de estar referenciándolos nuevamente. Por ejemplo:


h1 {
 font-size: 32px;

 @media (max-width: 350px) {
  font-size: 28px;
 }
}



Lo que se traduce en css como:


h1 {
 font-size: 32px;
}
@media (max-width: 350px) {
 h1 {
  font-size: 28px;
 }
}




Archivos parciales


Con CSS, si queremos separar nuestro código en diferentes archivos para tenerlo mejor organizado y clasificado, tenemos que luego linkear cada uno de estos archivos en nuestro html. Esto no es muy recomendado, ya que aumenta los tiempos de carga de nuestra página. Sass nos soluciona esto permitiendonos crear archivos parciales.

Estos archivos parciales no se transforman directamente en archivos .css, sino que tienen que ser primero importados en un archivos sass que no sea parcial, y compilar este último.
Al importar un archivo parcial, lo que estamos haciendo es incluir su código dentro del archivo al que lo estamos importando.

Para declarar un archivo parcial, basta con anteponerle al nombre del archivo un guión bajo, por ejemplo:


_variables.scss



Luego, para importar este archivo, es decir, para incluir su código, necesitamos utilizar la instrucción:


@import 'variables'



dentro del archivo al que lo queremos importar. Hay que prestar atención que cuando importamos, no ponemos el nombre del archivo ni con guión bajo, ni con la extensión.


La instrucción @import no toma el nombre de un archivo, sino la ruta relativa al mismo, por lo tanto, si por ejemplo nuestro archivo a importar está dentro de una carpeta general, tenemos que usar lo siguiente:


@import 'general/variables'




Organización


La posibilidad de separar nuestro código en distintos archivos nos permite organizarlo y distribuirlo de forma que sea más entendible y que nos facilite la tarea a la hora de encontrar y modificar estilos particulares. Una posible (no la única) forma de hacerlo, es la siguiente:


fonts
images
html
 about.html
 contact.html
styles
 general
  _variables.scss
  _fonts.scss
  _normalize.scss
 components
  _buttons.scss
  _inputs.scss
  _text.scss
 css
  main.css
 layout
  _header.scss
  _nav.scss
  _aside.scss
  _footer.scss
 pages
  _home.scss
  _about.scss
  _contact.scss
 main.scss
index.html



La estructura anterior cuenta con un único archivo scss completo, main.scss, que va a ser compilado a css. El resto de los archivos son parciales. En este archivo main.scss no se escribe nada de código, sólo sirve para importar los demás archivos. Dentro del mismo escribiríamos:


@import 'general/variables';
@import 'general/fonts';
@import 'general/normalize';

@import 'components/buttons';
...



Dentro de general irían las declaraciones más generales y que se utilizan en todos los estilos, por ejemplos las variables y el archivo de normalización.


En components van aquellos elementos que se repiten en distintas páginas, y que comparten ciertos estilos, por ejemplos, botones, inputs, cuadros de diálogo, algunos textos, etc. La idea es separarlos en archivos parciales por tipo, y que sólo refiramos a ellos a elementos o conjuntos de elementos html muy chiquitos o indivisibles, es decir, que ya no pueden ser subdivididos y van siempre juntos


En layout irían aquellas secciones más generales de las páginas que se repiten en varias o todas ellas. Ejemplos más usuales son el header, el footer, y el nav. Los incluimos dentro de un estilo archivo parcial cada uno.


Dentro de pages iría un archivo parcial por cada página, donde incluiríamos los estilos particulares a cada una de ellas, es decir, aquellas cosas específicas que o bien no se repiten en otras páginas, o son propias de ellas. Por ejemplo, un botón puede repetir sus estilos en múltiples páginas, pero la ubicación de dicho botón es propio de cada página en particular (no está siempre ubicado en el mismo lado).


Finalmente, dentro de css iría el archivo main.css que es la compilación de todos nuestros archivos, y que sería el que linkearíamos en nuestro html


Podemos aprovecharnos de la posibilidad de anidar selectores en sass para declarar los estilos que pertenecen a cada elemento, sección o página y que no afecten al resto de los elementos. Por ejemplo, dentro de _footer.scss incluiríamos algo así cómo:

footer {
 ul {
  li {

  }
 }
}



de modo tal que si tenemos algún ul li en alguna otra sección de nuestras páginas, no lo afecte los estilos del footer. Es decir, incluimos todos los estilos del footer dentro de un selector, de modo que dichos estilos sólo se apliquen a los que pertenecen a dicho elemento.


Lo mismo podemos hacer con las páginas y los layouts. Supongamos que los elementos h2 de la página principal tienen un tamaño de letra diferente al de los demás elementos h2 de las restantes páginas, entonces lo que podemos hacer es:

#home {
 h2 {
  font-size: 24px;
 }
}



de esta forma, estamos apuntando a los elementos h2 dentro de un elemento #home, y el resto de los elementos h2 no se verían afectados. Para que esto funcione, tendríamos que incluir dentro de nuestro html a todos los elementos de nuestra página home o index dentro de un contenedor con id home.
